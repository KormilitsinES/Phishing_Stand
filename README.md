## 📌 О проекте

**Phishing Stand** — это комплексное решение для автоматизированного развертывания изолированного стенда проведения фишинг-тестов. Проект объединяет 5 взаимосвязанных Docker-контейнеров, обеспечивающих полный цикл тестирования: от рассылки писем с DKIM-подписью до перехвата учетных данных через обратный прокси.

---

## 🏗️ Архитектура

Стенд состоит из 5 основных компонентов, объединенных в изолированную Docker-сеть:

1. **Gophish** - платформа для управления фишинговыми кампаниями.
2. **Evilginx2** - MITM-фреймворк для перехвата учетных данных, включая MFA.
3. **Milter** - фильтр для корректировки SMTP заголовков.
4. **Postfix** - почтовый сервер для отправки фишинговых сообщений.
5. **Nginx** - обратный прокси-сервер с SSL/TLS для приема входящих запросов.

## ✨ Возможности
- **Автоматизация**: развертывание "из коробки" с помощью одного скрипта.
- **Wildcard SSL**: генерация SSL сертификатов с помощью Let's Encrypt (DNS-01 challenge).
- **Доставляемость почты**: встроенная настройка Postfix + Milter и OpenDKIM для подписи исходящих писем.
- **Готовые DNS-шаблоны**: генерация SPF, DMARC, DKIM, TLS-RPT записей на основе предоставленной информации.
- **Миграция данных**: встроенные функции экспорта и импорта конфигурации для переноса стенда между серверами.
- **Пошаговый контроль**: возможность запуска, приостановки и возобновления процесса установки с любого шага.
- **Безопасность по умолчанию**: административные порты (Gophish, Evilginx2, Postfix) доступны только на 127.0.0.1.

## 📋 Требования
- VPS/VDS с публичным IP-адресом (рекомендуется: 2 CPU, 2 GB RAM, 20 GB HDD).
- Операционная система: Ubuntu 22.04 LTS.
- Доменное имя с возможностью управления DNS-записями (для настройки A, MX, TXT и _acme-challenge).
- Root-доступ к серверу для установки системных пакетов и управления Docker.
- Python 3: версия 3.8+ (в Ubuntu 22.04 присутствует по умолчанию).

## 🚀 Быстрый старт

### Шаг 1. Аренда и подготовка VPS
Разверните виртуальный сервер с Ubuntu 22.04 у любого надежного провайдера и подключитесь к нему по SSH.

### Шаг 2. Клонирование репозитория
Выполните следующие команды для обновления системы и загрузки исходного кода проекта:
```commandline
apt update && apt install git -y
git clone https://github.com/EfraimDiveroli-Pwn3d/Phishing_Stand.git
cd Phishing_Stand
```

### Шаг 3. Запуск автоматизации
Сделайте скрипт исполняемым и запустите его от имени суперпользователя:
```commandline
chmod +x management.py
sudo ./management.py
```
Следуйте подсказкам на экране для ввода доменных имен и настройки DNS.

## 📖 Подробная установка
Скрипт `management.py` выполняет 6 последовательных шагов. Вы можете проходить их автоматически или управлять процессом вручную (см. раздел [Режимы работы management.py](#-режимы-работы-managementpy)).
1. **install_deps**: проверка наличия и установка необходимых системных зависимостей (curl, gettext, docker.io, docker-compose-plugin).
2. **prepare_dirs**: создание структуры директорий для томов Docker (certs, gophish, evilginx2, postfix/dkim).
3. **configure_compose**: интерактивный опрос для ввода доменов и генерация финального docker-compose.yml.
4. **get_certs**: запуск Certbot в ручном режиме. Внимание: На этом этапе скрипт попросит вас добавить TXT-запись _acme-challenge у вашего DNS-провайдера. Для этого внесите запись, проверьте ее через dig и нажмите ENTER для продолжения.
5. **build_run**: сборка образов и запуск контейнеров (milter, postfix, nginx, evilginx2, gophish).
6. **print_info**: финальный вывод сгенерированных DNS-записей, URL-адресов и временного пароля администратора Gophish.

В процессе конфигурации доменов вам будет предложено ввести:

| Параметр       | Пример                  | Описание                   |
| -------------- | ----------------------- | -------------------------- |
| `BASE_DOMAIN`  | `my-domain.com`         | Базовый домен              |
| `TRACK_DOMAIN` | `track.my-domain.com`   | Домен для трекинга Gophish |
| `EVIL_DOMAIN`  | `phish.my-domain.com`   | Домен для Evilginx2        |
| `MX_DOMAIN`    | `mail.my-domain.com`    | Почтовый домен             |
| `ADMIN_EMAIL`  | `support@my-domain.com` | Email для Let's Encrypt    |

## ⚙️ Режимы работы management.py
Скрипт поддерживает гибкое управление процессом через аргументы командной строки.

| Аргумент           | Описание                                                         | Пример использования             |
| ------------------ | ---------------------------------------------------------------- | -------------------------------- |
| _(без аргументов)_ | Полное развертывание с 1-го по 6-й шаг.                          | `sudo ./management.py`           |
| `--from N`         | Начать выполнение с шага `N` (полезно при ошибках).              | `sudo ./management.py --from 4`  |
| `--only N`         | Выполнить исключительно шаг `N`.                                 | `sudo ./management.py --only 5`  |
| `--skip N M`       | Пропустить указанные шаги (например, если сертификаты уже есть). | `sudo ./management.py --skip 4`  |
| `--confirm`        | Запрашивать подтверждение `Y/n` перед каждым шагом.              | `sudo ./management.py --confirm` |
| `--reset`          | Очистить файл состояния (`.deploy_state`) и начать заново.       | `sudo ./management.py --reset`   |

## 💾 Экспорт и импорт конфигурации
Для удобства миграции стенда на другой сервер или создания резервных копий реализованы специальные режимы.

### Экспорт данных
Создает архив `phishing_stand_backup.tar.gz`, содержащий все критически важные данные:
```commandline
sudo ./management.py --export
```

#### Что включено в архив:
- База данных и конфигурационный файл Gophish.
- Данные сессий и phishlets Evilginx2.
- Сгенерированные DKIM-ключи и SSL-сертификаты Let's Encrypt.
- Файл переменных окружения .env и логи развертывания.

### Импорт данных
Восстанавливает состояние стенда из архива phishing_stand_backup.tar.gz (архив должен находиться в корневой папке проекта):
```commandline
sudo ./management.py --import
```

#### Сценарий миграции на новый сервер:
1. На старом сервере: `sudo ./management.py --export`.
2. Скопируйте `phishing_stand_backup.tar.gz` на новый сервер.
3. На новом сервере: клонируйте репозиторий, запустите `sudo ./management.py --import`, затем `sudo ./management.py --skip 4 --from 5` для перезапуска контейнеров с восстановленными данными.

### 🌐 DNS конфигурация
Для корректной работы стенда и попадания писем во "Входящие", после развертывания необходимо добавить следующие записи у вашего DNS-провайдера (скрипт выведет их автоматически):

| Тип   | Имя (Host)        | Значение (Value)                             | Назначение                               |
| ----- | ----------------- | -------------------------------------------- |------------------------------------------|
| `A`   | `@`               | `<IP_VPS>`                                   | Базовый домен                            |
| `A`   | `track`           | `<IP_VPS>`                                   | Домен для трекинга Gophish               |
| `A`   | `phish`           | `<IP_VPS>`                                   | Домен для фишинговых страниц (Evilginx2) |
| `A`   | `mail`            | `<IP_VPS>`                                   | Почтовый домен                           |
| `MX`  | `@`               | `mail.<your_domain>.com` (Priority: 10)      | Маршрутизация почты                      |
| `TXT` | `@`               | `v=spf1 a mx ip4:<IP_VPS> -all`              | SPF политика                             |
| `TXT` | `_dmarc`          | `v=DMARC1; p=quarantine; rua=mailto:rua@...` | DMARC отчетность                         |
| `TXT` | `mail._domainkey` | `v=DKIM1; h=sha256; k=rsa; p=...`            | DKIM подпись                             |
| `TXT` | `_smtp._tls`      | `v=STSv1; id=...`                            | SMTP TLS Reporting                       |

## 🔧 Полезные команды

### Управление контейнерами
```commandline
# Просмотр статуса всех контейнеров
docker compose ps

# Просмотр логов в реальном времени
docker compose logs -f

# Логи конкретного сервиса (например, postfix)
docker compose logs -f postfix

# Перезапуск одного из сервисов
docker compose restart gophish

# Полная остановка стенда
docker compose down

# Запуск стенда (без пересборки)
docker compose up -d
```

### Доступ к административной панели GoPhish
Порты проброшены только на localhost в целях безопасности. Используйте SSH-туннель:
```commandline
# В локальном терминале выполните:
ssh -L 3333:127.0.0.1:3333 root@<IP_VPS>

# Откройте в браузере:
https://127.0.0.1:3333
```

### Доступ к консоли Evilginx2
```
docker exec -it evilginx2 sh
./evilginx -developer -p phishlets/
```
